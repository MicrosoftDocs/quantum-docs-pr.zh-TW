# <a name="reporting-bugs"></a>報告錯誤 (Bug) #

如果您認為在「量子開發工具組」的元件中發現錯誤，我們非常感謝您的報告。
畢竟，其他人可能也會面臨相同的問題，讓我們知道可以協助我們為每個人進行修正。
報告 bug 的第一個步驟是從查看每個儲存機制上發現的現有問題開始著手，因為可能有其他人回報了相同的問題。

如果之前已發現問題，提供可協助我們診斷並解決問題的其他資訊或內容，可能會很有説明。
在此情況下，請隨意參與問題的討論。

如果您的 bug 未曾回報過，則我們會感謝您是否在有問題的元件的儲存機制上開啟新問題。
當建立新的問題時，系統會提示您提供一個範本，其中會建議一些通常有助於提出資訊性 bug 報告的資訊：

- 重現問題的步驟
- 您預期會發生的情況
- 實際發生的情況
- 軟體環境的相關資訊 (例如： OS、.NET Core 和量子開發工具組版本) 

您也可以[建立提取要求](https://help.github.com/articles/about-pull-requests/)來直接修正 bug，如果非常簡單，而且不值得討論 (例如，錯誤) 。
