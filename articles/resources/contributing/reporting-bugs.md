# <a name="reporting-bugs"></a>報告錯誤 (Bug) #

如果您認為您在量子開發工具組的元件中發現錯誤，我們很高興有一份報告。
畢竟，其他人可能也會面臨相同的問題。讓我們知道可協助我們為每個人進行修正。
報告 bug 的第一個步驟是從查看每個存放庫上發現的現有問題開始，因為有可能是其他人回報了相同的問題。

如果之前已發現問題，則提供其他資訊或內容有助於我們診斷並解決問題，可能會有所説明。
在這種情況下，請隨意參與此問題的討論。

如果之前未回報您的 bug，建議您在有問題的元件的存放庫上開啟新問題。
當您建立新的問題時，系統會提示您提供範本，以建議一些資訊，這些資訊通常有助於進行資訊性錯誤報表：

- 重現問題的步驟
- 您預期會發生的情況
- 實際發生的情況
- 軟體環境的相關資訊 (例如： OS、.NET Core 和量子開發工具組版本) 

您也可以 [建立提取要求](https://help.github.com/articles/about-pull-requests/) 來直接修正錯誤，如果非常直接，也不值得討論 (例如，錯誤) 。
