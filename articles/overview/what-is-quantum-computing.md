# <a name="what-is-quantum-computing"></a>什麼是量子運算？

世界上有某些問題極為困難，數量上不可思議的龐大，因此即使讓世界上的每一部超級電腦聯手處理這個問題，仍然要花上比宇宙的壽命還要長的時間才能解決。 量子電腦有望解決地球上的某些最大挑戰，環境方面、農業方面、健康方面、能源方面、氣候方面、材料科學方面，以及我們從未想到的各種方面。 量子電腦的影響將會非常深遠，其影響程度和 1947 年時，因為創造出電晶體，而成就了現今的數位經濟是一樣的。

量子運算會利用量子物理的獨特行為，來提供全新且功能巨大的運算模型。 量子物理的理論認為，量子級別的物質可以疊加多個傳統狀態。 這些狀態會彼此干擾，就和潮汐池中的波浪一樣。  假定測量之後物質的狀態會「塌縮」為其中一個古典狀態。 之後，重複相同的測量將產生相同的古典結果。  當粒子間的互動會讓各自的量子狀態無法完全獨立於其他粒子單獨描述，即使這些粒子實際上距離遙遠，就會發生量子纏結。  

量子運算會以物質的量子狀態儲存資訊，並使用量子的疊加和纏結本質透過計算該資訊實行量子操作，進而利用並學習量子干擾的程式設計。

量子運算聽起來可能會讓人退避三舍，但只要有適當的資源，您馬上就可以開始建置量子應用程式。

## <a name="the-qubit"></a>量子位元

量子運算定義了反映量子行為的運算概念。  量子運算從量子位元的概念開始。  在量子運算中，量子位元 (**qubit**) 是量子資訊的單位，就和傳統位元一樣。 其中，古典位元會有單一二進位值，例如 0 或 1，而量子位元的狀態則可以同時是 0 和 1 的**疊加**。  

測量量子位元的舉動會改變量子位元的狀態。 測量時，量子位元會從疊加狀態變為兩個古典狀態的其中一個。  

多個量子位元也可以相互**纏結**。 當我們測量一個纏結的量子位元時，對其他量子位元狀態的了解也會隨之更新。

## <a name="quantum-algorithms"></a>量子演算法

量子演算法是設計成利用量子的本質和行為來加速古典演算法，也可以設計成提供全新的實體系統建模方式。  這些演算法會利用量子位元資訊的編碼方式，以及對疊加狀態中多個纏結的量子位元進行操作的平行特性。  

古典電腦以位元為單位將資訊編碼；每個位元編碼有 2 個可能的值，0 或 1。  一個量子位元會同時對 0 和 1 兩個值進行編碼。  兩個古典位元的編碼是 4 個可能值 (00、01、10、11) 的其中一個，而 2 個量子位元的編碼則是這 4 個值同時存在的任何疊加，雖然我們在測量時只會得到這些值的其中一個。  四個古典位元的編碼是 16 個值的其中一個。  四個量子位元的編碼則是16 個值同時存在的任何疊加，依此類推，呈指數增加。  100 個量子位元可以編碼的資訊，比現今最大的電腦系統能提供的還要多。  

除此之外，當多個纏結的量子位元以相干方式行動時，便可同時處理多個選項。 這會讓這些量子位元在處理資訊時，所需時間遠遠少於最快的非量子系統。

運用這些量子特性已經是量子演算法研究數十年來追求的目標，而且已經找到許多可以解決問題的創新技術，花費的時間更是解決傳統方式的數分之一。  

最著名的量子演算法之一是用來進行因數分解的「Shor 演算法」  ，此演算法讓傳統上非常棘手，要將巨大數字因數分解為兩個質數的問題，變得很快就能得出結果，因此讓傳統的密碼編譯受到挑戰。

從比較積極的角度來看，可供安全散發密碼編譯金鑰的演算法會因為量子位元的疊加、量子糾纏和**不可複製** (也就是，無法在不被偵測到的情況下複製量子位元) 特性而有可能成真。

_格羅弗演算法_強調一種量子演算法技術，可讓您在搜尋非結構化資料時進行二次加速。


## <a name="quantum-hardware"></a>量子硬體

在傳統電腦中，位元會對應到晶片電路中的電壓位準。 量子運算硬體可透過許多不同的量子位元表現形式來實現：捕獲離子、超導性質、中性原子、電子自旋、光偏極化、拓撲量子位元。 量子硬體是一種新興技術。 量子位元天生就很脆弱，並且會因為與環境互動而讓相干程度變差。 這需要透過延展性來平衡系統的準確性。 規模越大 (也就是量子位元數目越多)，錯誤率就會越高。

Microsoft 正在根據拓撲量子位元來開發量子電腦。 我們認為拓撲量子位元較不會受到其環境變更所影響，因此可減少外部糾錯程度。 拓撲量子位元提高了穩定性和對抗環境雜訊的能力，這表示它們可以更輕鬆地調整規模，並長期保持可靠。

## <a name="quantum-computing--a-full-hardware-and-software-stack"></a>量子運算 – 完整的硬體和軟體堆疊 

Microsoft 的量子計畫的獨特之處在於，我們著重在調整系統的每一個元件，以提供真正的量子影響。 這種全方位的方法涉及： 
* 使用可靠、可擴充且容錯的拓撲量子位元來建立量子電腦， 
* 設計獨特的低溫控制平面，消耗電力和散熱極低， 
* 開發完整的軟體堆疊，讓量子電腦的程式設計和大規模系統的控制得以實現。 

所以我們推出了開放原始碼 Quantum Development Kit (QDK) 來讓您更容易進行量子程式設計和開發演算法。 我們的高階程式設計語言 Q# 解決了量子程式設計的挑戰。  我們將 Q# 設計為以量子為中心、著重於演算法和應用程式開發的高階程式語言。 Q# 編譯器已整合到軟體堆疊中，因此您可以將量子演算法編譯到量子電腦的基本操作中。  達到一定規模 (量子位元數目) 後，便可用傳統電腦來模擬量子運算。 透過模擬，您可以馬上開始編寫量子程式，以在日後的量子硬體上執行。  我們也同時提升 Q# 的範例、程式庫和學習體驗，讓您可以輕鬆地立即開始量子程式設計。
