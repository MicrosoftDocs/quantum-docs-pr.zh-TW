### <a name="minibatch-size"></a><span data-ttu-id="cad61-120">迷你批次大小</span><span class="sxs-lookup"><span data-stu-id="cad61-120">Minibatch size</span></span>

### <a name="training-epochs-tolerance-gridlocks"></a><span data-ttu-id="cad61-124">訓練 epoch、容錯、gridlocks</span><span class="sxs-lookup"><span data-stu-id="cad61-124">Training epochs, tolerance, gridlocks</span></span>

### <a name="measurements-count"></a><span data-ttu-id="cad61-132">測量計數</span><span class="sxs-lookup"><span data-stu-id="cad61-132">Measurements count</span></span>

<span data-ttu-id="cad61-133">在配量裝置數量上預估訓練/驗證分數和隨機漸層的元件，以估計需要多次測量適當可預見值的量子狀態重迭。</span><span class="sxs-lookup"><span data-stu-id="cad61-133">Estimating the training/validation scores and the components of the stochastic gradient on a quantum device amounts to estimating quantum state overlaps that requires multiple measurements of the appropriate observables.</span></span> <span data-ttu-id="cad61-134">量值的數目應該調整為 $O （1/\ epsilon ^ 2） $，其中 $ \epsilon $ 是所需的估計錯誤。</span><span class="sxs-lookup"><span data-stu-id="cad61-134">The number of measurements should scale as $O(1/\epsilon^2)$ where $\epsilon$ is the desired estimation error.</span></span>
<span data-ttu-id="cad61-135">根據經驗法則，初始量測計可能大約是 $ 1/\ .mbox {容錯} ^ 2 $ （請參閱上一個段落中的容錯定義）。</span><span class="sxs-lookup"><span data-stu-id="cad61-135">As a rule of thumb, the initial measurements count could be approximately $1/\mbox{tolerance}^2$ (see definition of tolerance in the previous paragraph).</span></span> <span data-ttu-id="cad61-136">如果梯度下降變得太不穩定，而且收斂過於難以達成，則需要將測量計數向上修改。</span><span class="sxs-lookup"><span data-stu-id="cad61-136">One would need to revise the measurement count upward if the gradient descent appears to be too erratic and convergence too hard to achieve.</span></span>

<span data-ttu-id="cad61-138">這是分類器定型公用程式的機率函數非常少使用，這表示它在參數空間中通常會有許多本機 optima，品質可能會有顯著的差異。</span><span class="sxs-lookup"><span data-stu-id="cad61-138">The likelihood function which is the training utility for the classifier is very seldom convex, meaning that it usually has a multitude of local optima in the parameter space that may differ significantly by quality.</span></span> <span data-ttu-id="cad61-139">由於 SGD 程式只能收斂到一個特定的最佳方式，因此請務必探索多個起始參數向量。</span><span class="sxs-lookup"><span data-stu-id="cad61-139">Since the SGD process can converge to only one specific optimum, it is important to explore multiple starting parameter vectors.</span></span> <span data-ttu-id="cad61-140">機器學習服務中的常見做法是隨機初始化這類啟動向量。</span><span class="sxs-lookup"><span data-stu-id="cad61-140">Common practice in machine learning is to initialize such starting vectors randomly.</span></span> <span data-ttu-id="cad61-141">Q # 訓練 API 接受這類啟動向量的任意陣列，但基礎程式碼會依序進行探索。</span><span class="sxs-lookup"><span data-stu-id="cad61-141">The Q# training API accepts an arbitrary array of such starting vectors but the underlying code explores them sequentially.</span></span> <span data-ttu-id="cad61-142">在多核心電腦上，或實際上是在任何平行運算架構上，建議您在呼叫時，與不同的參數初始化平行執行 Q # 訓練 API 的數個呼叫。</span><span class="sxs-lookup"><span data-stu-id="cad61-142">On a multicore computer or in fact on any parallel computing architecture it is advisable to perform several calls to Q# training API in parallel with different parameter initializations across the calls.</span></span>
